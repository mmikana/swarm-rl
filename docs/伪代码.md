# PPO-CBF 模式 B 伪代码（与 Sample Factory APPO 一致）

## 符号说明

| 符号 | 含义 | 代码变量 | 说明 |
|------|------|---------|------|
| $u_t^{RL}$ | RL 标称动作 | `u_rl` | Policy 网络直接输出的动作 |
| $u_t^{safe}$ | CBF 修正后的安全动作 | `u_safe` | 经过 CBF-QP 层修正的动作 |
| $u_t^{final}$ | 最终执行动作 | `u_final` | **实际输出给环境的动作** |
| $\pi_\phi$ | 策略网络 | `policy_head` | - |
| $V_{\theta}$ | Critic 网络 | `critic_head` | 输出状态价值 V(s) |
| $\log \pi$ | 对数概率 | `log_probs` | 基于 $u_{RL}$ 计算 |

---

## 轨迹收集阶段 (Rollout Phase)
对于每个时间步 $t = 1, \dots, T$：

**1. 策略采样**：Actor 根据当前状态采样标称动作
$$u_t^{RL} \sim \pi_\phi(s_t)$$

**2. 安全修正**：通过可微 QP 层求解执行动作
$$u_t^{safe} = \text{QP}(u_t^{RL}, A_t, b_t)$$
$$u_t^{final} = u_t^{safe}$$

注：$b_t$ 中包含了角速度 Risk 项 $k_\omega \|\omega_{xy}\|^2$ 的物理补偿。

**3. 计算概率**：计算并存储对数概率
$$\log \pi_\phi(u_t^{RL} | s_t)$$
作为后续更新的 $\log \pi_{old}$。

**4. 环境交互**：执行 $u_t^{final}$，获得任务奖励 $r_t$ 和下一状态 $s_{t+1}$。

**5. 价值评估**：Critic 评估当前状态价值
$$V_t = V_\theta(s_t)$$

**6. 存储数据**：将 $(s_t, u_t^{final}, V(s_t), \log \pi_\phi, r_t)$ 存入 Buffer。

---

## 优势与价值计算 (GAE & Value Estimation)

### 标准 PPO V(s) 计算（无 Q 值）

**Critic 输出**：
$$V(s) = \text{CriticHead}(\text{encoder\_out})$$

**注意**：PPO 使用 V(s)，**不使用 Q(s,u)**！

### TD 误差计算

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

其中：
- $r_t$：环境奖励（执行 $u_t^{final}$ 后获得）
- $V(s_{t+1})$：下一状态的价值（目标网络或主网络）
- $V(s_t)$：当前状态的价值

### GAE 优势估计

$$\hat{A}_t = \delta_t + (\gamma \lambda) \delta_{t+1} + \dots + (\gamma \lambda)^{T-t-1} \delta_{T-1}$$

## 参数优化阶段 (Optimization Phase)

在多个训练 Epoch 中通过反向传播更新：

### Actor 损失函数 (PPO-CLIP)

$$L^{CLIP}(\phi) = \hat{\mathbb{E}}_t \left[ \min \left( \frac{\pi_\phi(u_t^{RL}|s_t)}{\pi_{old}(u_t^{RL}|s_t)} \hat{A}_t, \text{clip} \left( \frac{\pi_\phi(u_t^{RL}|s_t)}{\pi_{old}(u_t^{RL}|s_t)}, 1-\epsilon, 1+\epsilon \right) \hat{A}_t \right) \right]$$

**梯度流路径**：
$$\nabla_\phi L^{CLIP} \leftarrow \frac{\partial \log \pi_\phi}{\partial \phi}$$

**注意**：
- PPO 的 Actor Loss 基于 $\log \pi(u_t^{RL}|s_t)$，**不直接依赖 QP 层梯度**
- 安全性通过**隐式学习**：如果 $u_t^{RL}$ 经常被 CBF 修正，导致奖励降低，Policy 会学会避免这类动作

### Critic 损失函数

$$L(\theta) = \hat{\mathbb{E}}_t \left[ \frac{1}{2} \left( V_\theta(s_t) - (r_t + \gamma V(s_{t+1})) \right)^2 \right]$$

**物理意义**：Critic 学习预测当前状态 $s_t$ 的期望回报，**不依赖具体动作**。

## 核心要点总结

### 1. PPO 使用 V(s)，不使用 Q(s,u)

| 组件 | 输入 | 输出 | 依赖动作 |
|------|------|------|---------|
| **Critic** | `encoder_out` | `V(s)` | ❌ 否 |
| **Policy** | `encoder_out` | `u_rl` | ✅ 是 |

### 2. CBF 的作用

- **训练时**：$u_t^{final} = u_t^{safe}$，保证探索安全性
- **推理时**：$u_t^{final} = u_t^{safe}$，保证部署安全性
- **学习机制**：隐式通过 reward 信号，而非显式梯度

### 3. 与 Modular Value Update 的区别

| 特性 | 标准 PPO | Modular Value Update |
|------|---------|-------------------|
| Critic 输出 | $V(s)$ | $Q(s, u)$ |
| TD Target | $r + \gamma V(s')$ | $r + \gamma Q(s', u_{rl}')$ |
| 零样本迁移 | ❌ 不支持 | ✅ 支持 |
| 实现复杂度 | 低 | 高 |

**我们的方案**：使用标准 PPO，**不实现 Modular Value Update**，保持与 Sample Factory 完全兼容。

## 理论推导（标准 PPO + CBF）

### PPO 目标函数

$$\max_\phi \mathbb{E}_{s \sim \rho_\pi, u \sim \pi_\phi} \left[ \sum_{t=0}^\infty \gamma^t r(s_t, u_t^{final}) \right]$$

其中 $u_t^{final} = \text{QP}(u_t^{RL})$。

### 梯度估计

$$\nabla_\phi J = \mathbb{E} \left[ \sum_{t=0}^\infty \nabla_\phi \log \pi_\phi(u_t^{RL}|s_t) \cdot A_t \right]$$

**关键**：梯度通过 $\log \pi_\phi(u_t^{RL}|s_t)$ 回传，**不通过 QP 层**。

### 安全性保证

$$u_t^{final} = \arg\min_u \|u - u_t^{RL}\|^2 \quad \text{s.t.} \quad A_t u \ge b_t$$

CBF 保证 $u_t^{final}$ 始终在安全区域内，但**不改变 PPO 的优化目标**。

## 总结

### 正确的 PPO + CBF 设计

1. ✅ **Critic 输出 V(s)**：不依赖动作，标准 PPO
2. ✅ **Actor 输出 u_rl**：通过 CBF-QP 修正为 $u_{safe}$
3. ✅ **环境执行 u_safe**：保证安全性
4. ✅ **GAE 计算 Advantage**：基于 V(s) 和 rewards
5. ✅ **Actor Loss 基于 log π**：不直接依赖 QP 梯度
6. ✅ **安全性通过 reward 隐式学习**：Policy 学会避免被大幅修正的动作

### 与 `伪代码_samplefactory.md` 的一致性

- ✅ Critic 输出：`V(s)`（不是 `Q(s,u)`）
- ✅ Actor Loss：PPO-CLIP（基于 `log π`）
- ✅ Critic Loss：MSE（基于 `V(s)` 和 `returns`）
- ✅ GAE 计算：标准公式
- ✅ 网络结构：与 baseline 完全一致
